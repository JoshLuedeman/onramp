# OnRamp API Reference

Base URL: `/api`

> **Interactive API documentation** is auto-generated by FastAPI:
>
> | Tool | URL | Description |
> |------|-----|-------------|
> | Swagger UI | `/docs` | Interactive API explorer — try endpoints directly |
> | ReDoc | `/redoc` | Read-only API reference with complete schemas |
>
> The OpenAPI spec is available at `/openapi.json`.

## Authentication

All endpoints require Entra ID authentication via Bearer token in the `Authorization` header:

```
Authorization: Bearer <access-token>
```

In development mode (when `ONRAMP_AZURE_TENANT_ID` is not set), a mock user is automatically provided — no token is required.

### Roles

| Role | Permissions |
|------|-------------|
| Admin | Full access including tenant management and audit |
| Architect | Create/edit projects, generate architectures, deploy |
| Viewer | Read-only access to projects and architectures |

## API Versioning

The API supports version headers. Requests to `/api/v1/...` are rewritten to `/api/...` automatically. The `X-API-Version` header is included in all responses.

## Common Response Patterns

### Success

```json
{ "id": "uuid", "status": "success", "data": { } }
```

### Error

```json
{ "detail": "Error message describing what went wrong" }
```

### Pagination

List endpoints support pagination via query parameters:

```
GET /api/projects/?page=1&per_page=20
```

### Status Codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `201` | Created |
| `202` | Accepted (async operation started) |
| `400` | Bad request (validation error) |
| `401` | Unauthorized (missing or invalid token) |
| `403` | Forbidden (insufficient role) |
| `404` | Not found |
| `409` | Conflict (optimistic concurrency violation) |
| `422` | Unprocessable entity (Pydantic validation failure) |
| `429` | Rate limited |
| `500` | Internal server error |
